# Privacy Policy

Effective date: March 8, 2026

This Privacy Policy describes how copilot-remote-control handles information when the Discord bot, relay server, and VS Code extension are used.

## 1. Summary

copilot-remote-control is designed to route prompts from Discord to a local VS Code instance running GitHub Copilot.

The project does not include advertising, analytics, or data brokerage features.

## 2. Information that may be processed

Depending on how the project is deployed, the following information may be processed:

- Discord user display names and usernames
- Discord server, channel, thread, and message identifiers
- prompt content submitted through the Discord bot
- response content returned from GitHub Copilot
- technical connection data such as client identifiers and relay status messages
- local configuration values such as relay URL and client ID
- logs generated by the bot, relay server, or VS Code extension

## 3. How information is used

Information is used only to:

- route prompts from Discord to the connected VS Code instance
- return Copilot responses back to Discord
- operate, debug, and secure the bot, relay, and extension
- investigate failures, abuse, or misuse

## 4. Data storage and retention

The project does not include a dedicated long-term database by default.

However, data may still appear in:

- Discord messages and Discord logs
- local terminal output and application logs
- GitHub Copilot or other third-party service processing
- any hosting, monitoring, or logging system configured by the operator

Retention depends on the environment in which the project is run.

## 5. Third-party services

This project relies on third-party platforms and software, including Discord, GitHub, GitHub Copilot, and Visual Studio Code.

Those services may process information under their own privacy policies and terms.

## 6. Data sharing

The project does not intentionally sell personal data.

Information is shared only as necessary to make the system function, including transmission between Discord, the relay server, the VS Code extension, and GitHub Copilot.

## 7. Security

Reasonable efforts may be made to protect credentials and access, but no system can be guaranteed to be completely secure.

Operators are responsible for protecting bot tokens, secrets, hosts, and deployment environments.

## 8. User choices

If you do not want your prompts or related metadata processed through this system, do not use the bot or service.

If you operate the service, you are responsible for handling any user requests that apply to your deployment.

## 9. Changes to this policy

This policy may be updated from time to time.

Continued use of the service after updates are published means you accept the updated policy.
